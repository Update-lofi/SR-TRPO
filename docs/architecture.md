# Архитектура проекта

## Схема взаимодействия
Клиент (Браузер) -> Веб-сервер (Flask) -> Хранилище (SQLite)

## Описание
1. **Клиент:** Отвечает за отображение интерфейса (HTML/CSS, Bootstrap) и отправку форм.
2. **Веб-сервер:** Flask-приложение обрабатывает маршруты, валидирует данные форм и реализует бизнес-логику (например, проверку ответов в режиме тренировки).
3. **Хранилище:** Реляционная база данных SQLite через ORM (SQLAlchemy). Выбрана SQLite, так как приложение не требует сложной настройки сервера БД, подходит для прототипа.
